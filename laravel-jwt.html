JWT Authentication in Laravel API - Complete Beginner's Guide

Authentication is one of the most important features of any web application. Whenever users log in to an application, the server needs a secure way to verify their identity for every future request.

For traditional Laravel web applications, Laravel uses sessions and cookies. However, when building REST APIs for mobile applications, React, Vue, Angular, Flutter, or other frontend frameworks, session-based authentication isn't the best option.

This is where JWT (JSON Web Token) comes into the picture. JWT allows the server to authenticate users without storing session data, making it perfect for RESTful APIs.

Prerequisites

Before starting, make sure you have:

  • PHP 8.2 or higher
  • Composer installed
  • Laravel 10, 11, or 12
  • MySQL Database
  • Postman

What is JWT?

JWT stands for JSON Web Token. It is an open standard used to securely transmit information between two parties as a JSON object.

Whenever a user logs in successfully, the server generates a unique token. The client stores this token and sends it with every future API request.

Important: The server does not need to remember the logged-in user. Everything required is stored inside the JWT token itself.

Step 1 - Create Laravel Project

composer create-project laravel/laravel jwt-api

cd jwt-api

Step 2 - Install JWT Package

Install the popular JWT package.

composer require tymon/jwt-auth

Step 3 - Publish Configuration

php artisan vendor:publish --provider="Tymon\JWTAuth\Providers\LaravelServiceProvider"

This command creates:

config/jwt.php

Step 4 - Generate Secret Key

php artisan jwt:secret

Your .env file will now contain:

JWT_SECRET=your_secret_key

Step 5 - Configure Database

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=jwt_api
DB_USERNAME=root
DB_PASSWORD=
Run migrations. 
php artisan migrate

Step 6 - Configure Authentication Guard

Edit config/auth.php

'guards' => [

    'api' => [
        'driver' => 'jwt',
        'provider' => 'users',
    ],

],

Step 7 - Update User Model

Implement JWTSubject interface.

use Tymon\JWTAuth\Contracts\JWTSubject;

class User extends Authenticatable implements JWTSubject
{
    public function getJWTIdentifier()
    {
        return $this->getKey();
    }

    public function getJWTCustomClaims()
    {
        return [];
    }
}

Step 8 - Create Authentication Controller

php artisan make:controller Api/AuthController

Create the following methods:

  • register()
  • login()
  • profile()
  • refresh()
  • logout()

Step 9 - Register User

Endpoint:

POST /api/register

Request:

{
    "name":"John",
    "email":"john@example.com",
    "password":"password"
}

Step 10 - Login

POST /api/login

If credentials are correct, Laravel returns:

{
    "access_token":"eyJhbGc...",
    "token_type":"Bearer",
    "expires_in":3600
}
Congratulations! The user is now authenticated.

Step 11 - Protect Routes

Add middleware:

Route::middleware('auth:api')->group(function(){

    Route::get('/profile',[AuthController::class,'profile']);

});

Step 12 - Send Token

Every request should include:

Authorization: Bearer eyJhbGc....

Step 13 - Get Logged-in User

Request:

GET /api/profile

Response:

{
    "id":1,
    "name":"John",
    "email":"john@example.com"
}

Step 14 - Refresh Token

POST /api/refresh

A new JWT token will be generated.

Step 15 - Logout

POST /api/logout

The current token becomes invalid.

Advantages of JWT Authentication

Feature Benefit
Stateless No server-side session storage required.
Fast Every request contains authentication information.
Secure Digitally signed tokens prevent tampering.
Cross Platform Works with React, Vue, Angular, Flutter, Android, and iOS.
Scalable Ideal for distributed microservices and APIs.

Common Errors and Solutions

JWT_SECRET not set

php artisan jwt:secret

Unauthenticated

Check that the Authorization header is correctly set: 

Authorization: Bearer your_token

Token Expired

Use the refresh endpoint to obtain a new access token.

Guard Not Working

Ensure the api guard uses the jwt driver and clear the configuration cache: 

php artisan config:clear
php artisan cache:clear

Best Practices

  • Always use HTTPS in production.
  • Never expose your JWT secret key.
  • Set an appropriate token expiration time.
  • Validate all incoming requests.
  • Use Laravel Form Requests for validation.
  • Store sensitive data securely.
  • Return meaningful API error responses.

Conclusion

JWT Authentication is one of the most widely used authentication methods for REST APIs in Laravel. Unlike session-based authentication, JWT is stateless, making it an excellent choice for SPAs, mobile applications, and microservices. By following this guide, you've learned how to create a Laravel API project, install and configure the JWT package, register users, authenticate logins, protect routes, retrieve authenticated user information, refresh expired tokens, and securely log users out.

As you continue building APIs, consider adding features like role-based authorization, email verification, password reset, and refresh token rotation to make your application even more secure and production-ready.
