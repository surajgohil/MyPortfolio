🎭 Traits — Reuse Without Inheritance
trait Greetable { public function greet() { return "Hello, I am " . $this->name; } } class User { use Greetable; // ← Inject trait public $name = "Alice"; } $u = new User(); echo $u->greet(); // Hello, I am Alice
✅ Traits solve the diamond problem — a class can use multiple traits!
❓ Q: What is the difference between static and instance methods?
Static methods belong to the CLASS and are called with ClassName::method(). They cannot access $this. Instance methods belong to an OBJECT and are called with $obj->method(). They can access $this.

🛡️ Prepared Statements (Prevent SQL Injection)
// ❌ DANGEROUS — SQL Injection risk! $sql = "SELECT * FROM users WHERE id = $_GET[id]"; // ✅ SAFE — Prepared Statement with PDO $pdo = new PDO("mysql:host=localhost;dbname=mydb", $user, $pass); $stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id"); $stmt->bindParam(':id', $_GET['id'], PDO::PARAM_INT); $stmt->execute(); $user = $stmt->fetch(PDO::FETCH_ASSOC);
📊 CRUD Operations
// CREATE $stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:n, :e)"); $stmt->execute([':n' => 'Alice', ':e' => 'alice@ex.com']); // READ $rows = $pdo->query("SELECT * FROM users")->fetchAll(); // UPDATE $pdo->prepare("UPDATE users SET name=:n WHERE id=:id")->execute([...]); // DELETE $pdo->prepare("DELETE FROM users WHERE id=:id")->execute([':id' => 5]);

Security 🔴 Must Know
SQL Injection, XSS, CSRF & Password Hashing
💉 SQL Injection Prevention
Never trust user input! Always use prepared statements or parameterized queries.
// ✅ Prevention with PDO Prepared Statement $stmt = $pdo->prepare("SELECT * FROM users WHERE email=?"); $stmt->execute([$_POST['email']]);
🖥️ XSS (Cross-Site Scripting) Prevention
// ❌ Vulnerable — outputs raw HTML echo $_GET['name']; // ✅ Safe — escape HTML entities echo htmlspecialchars($_GET['name'], ENT_QUOTES, 'UTF-8'); // Also sanitize inputs $clean = filter_input(INPUT_GET, 'name', FILTER_SANITIZE_STRING);
🔐 Password Hashing
// ✅ Hash password before storing $hash = password_hash($password, PASSWORD_BCRYPT); // ✅ Verify on login if (password_verify($inputPassword, $hash)) { echo "Login successful!"; } // ❌ NEVER do this: $hash = md5($password); // md5/sha1 are NOT secure for passwords!
🛡️ CSRF Protection
// Generate token and store in session $_SESSION['csrf'] = bin2hex(random_bytes(32)); // In form: <input type="hidden" name="csrf" value="<?= $_SESSION['csrf'] ?>"> // On submit — verify token: if (!hash_equals($_SESSION['csrf'], $_POST['csrf'])) { die('CSRF attack detected!'); }
⭐ Use hash_equals() instead of == to prevent timing attacks!

Error Handling 🟠 Important
Exceptions, try-catch, custom errors
⚠️ PHP Error Types
Fatal Error
Stops script execution immediately. E.g., calling undefined function.
Warning
Script continues but something is wrong. E.g., include missing file.
Notice
Minor issue, script continues. E.g., accessing undefined variable.
Parse Error
Syntax error found during compilation. Script won't run at all.
🎯 Try / Catch / Finally
try { if ($age < 0) { throw new InvalidArgumentException("Age cannot be negative"); } echo "Valid age: " . $age; } catch (InvalidArgumentException $e) { echo "Caught: " . $e->getMessage(); } catch (Exception $e) { echo "General error: " . $e->getMessage(); } finally { echo "This ALWAYS runs!"; // Cleanup code here }
🏗️ Custom Exception
class DatabaseException extends RuntimeException { public function __construct($msg, $code = 0) { parent::__construct("DB Error: " . $msg, $code); // Log to file, notify admin, etc. } } throw new DatabaseException("Connection failed", 500);
❓ Q: What is the difference between Exception and Error in PHP 7+?
Exception is for application-level errors you expect. Error is for engine-level failures (type errors, parse errors). Both implement Throwable interface — you can catch both with catch(Throwable $t).

Arrays & Functions 🟠 Important
Array functions, closures, and arrow functions
🗃️ Key Array Functions
$nums = [1, 2, 3, 4, 5]; // array_map — transform each element $doubled = array_map(fn($n) => $n * 2, $nums); // [2,4,6,8,10] // array_filter — keep matching elements $evens = array_filter($nums, fn($n) => $n % 2 === 0); // [2,4] // array_reduce — reduce to single value $sum = array_reduce($nums, fn($carry, $item) => $carry + $item, 0); // 15 // usort — custom sorting usort($arr, fn($a, $b) => $a['age'] <=> $b['age']); // Other useful functions array_unique($arr); // remove duplicates array_merge($a, $b); // merge arrays in_array(3, $nums); // check if value exists array_key_exists('k', $arr); // check if key exists
🔮 Closures & Arrow Functions
// Closure (anonymous function) $multiply = function($x, $y) { return $x * $y; }; // Use 'use' to capture outer variables $factor = 3; $triple = function($n) use ($factor) { return $n * $factor; }; // Arrow function (PHP 7.4+) — auto-captures $factor $triple = fn($n) => $n * $factor; echo $triple(5); // 15
⭐ Arrow functions automatically capture outer scope — no need for use!
📦 Pass by Value vs Reference
// Pass by VALUE — original unchanged function addTen($n) { $n += 10; } $x = 5; addTen($x); echo $x; // 5 // Pass by REFERENCE — original changed function addTenRef(&$n) { $n += 10; } $y = 5; addTenRef($y); echo $y; // 15

Top Interview Q&A 🔴 Practice These!
Most commonly asked PHP interview questions with answers
❓ Q1: What is the difference between == and ===?
== (loose comparison) converts types before comparing: "1" == 1 is true. === (strict comparison) checks both value AND type: "1" === 1 is false. Always prefer === for predictable results.
❓ Q2: What are magic methods? Give examples.
Magic methods are predefined methods starting with double underscore that PHP calls automatically. Examples: __construct() on object creation, __toString() when echoing an object, __get()/__set() for inaccessible properties, __call() for undefined methods.
❓ Q3: Interface vs Abstract Class — when to use which?
Use Interface when you want to define a contract (what a class MUST do) without any implementation. A class can implement multiple interfaces. Use Abstract Class when you have shared implementation code — it can have method bodies, properties, and constructor. A class can only extend one abstract class.
❓ Q4: How do you prevent SQL injection in PHP?
Use prepared statements with PDO or MySQLi. Never concatenate user input directly into SQL queries. Use $pdo->prepare() with named parameters (:param) and bind values with bindParam() or pass them in execute([]).
❓ Q5: What is a Trait and why use it?
A Trait is a mechanism for code reuse in single-inheritance languages. It lets you "inject" methods into classes that don't share an inheritance hierarchy. A class can use multiple traits. Useful for cross-cutting concerns like logging, timestamps, soft-delete behavior.
❓ Q6: Session vs Cookie — explain the difference.
Sessions store data on the SERVER — more secure, limited by server memory. Cookies store data on the CLIENT's browser — less secure, size limit ~4KB, can persist across browser restarts with expiry. Sessions use a cookie (session ID) to identify the user.
❓ Q7: What is the difference between array_map and array_filter?
array_map() transforms every element and returns a new array of the same size. array_filter() keeps only elements where the callback returns true, returning a smaller subset array.
