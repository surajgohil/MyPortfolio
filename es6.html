let & const

Block-scoped variable declarations that replaced var

ES2015

📖 Explanation

Before ES6, JavaScript only had var — which is function-scoped and gets hoisted to the top of the function. ES6 introduced let and const which are block-scoped, meaning they only exist within the {} block they're declared in.

  • let — block-scoped, can be reassigned, cannot be re-declared in same scope
  • const — block-scoped, cannot be reassigned, must be initialized
  • var — function-scoped, hoisted, avoid in modern JS
  • const does NOT make objects/arrays immutable — it only prevents reassignment of the binding
JS — let
// let — reassignable, block-scoped let score = 0; score = 10; // ✅ OK if (true) { let blockVar = "inside"; console.log(blockVar); // "inside" } // console.log(blockVar) → ReferenceError! // Loop counter — classic let use case for (let i = 0; i < 3; i++) { console.log(i); // 0, 1, 2 }
JS — const
// const — cannot be reassigned const PI = 3.14159; // PI = 4 → TypeError! // Objects/arrays ARE mutable with const const user = { name: "Alice" }; user.name = "Bob"; // ✅ OK // user = {} → TypeError! const nums = [1, 2, 3]; nums.push(4); // ✅ OK console.log(nums); // [1,2,3,4]
⚠️ Temporal Dead Zone (TDZ): Unlike var, let and const are NOT accessible before their declaration in the code, even though they're "hoisted". Accessing them before declaration throws a ReferenceError.

Arrow Functions

Concise function syntax with lexical this binding

ES2015

📖 Explanation

Arrow functions provide a shorter syntax for writing functions. The key difference from regular functions is that arrow functions do not have their own this — they inherit this from the surrounding (lexical) scope. This makes them ideal for callbacks and methods that need to access the outer context.

  • No this, arguments, super, or new.target binding
  • Cannot be used as constructors (no new)
  • If body is a single expression, return is implicit
  • Wrap object literals in () for implicit return: () => ({ key: val })
JS — Arrow Function Syntax Variants
// Traditional function function add(a, b) { return a + b; } // Arrow — multi param, expression body (implicit return) const add = (a, b) => a + b; // Arrow — single param (no parens needed) const double = x => x * 2; // Arrow — no params const greet = () => "Hello!"; // Arrow — block body (explicit return needed) const multiply = (a, b) => { const result = a * b; return result; }; // Return object literal — wrap in () const makeUser = name => ({ name, id: Math.random() }); // Lexical this — arrow preserves outer context class Timer { constructor() { this.count = 0; } start() { setInterval(() => { // arrow: this = Timer this.count++; console.log(this.count); }, 1000); } } // Array methods — perfect use of arrows const nums = [1,2,3,4,5]; const evens = nums.filter(n => n % 2 === 0); const doubled = nums.map(n => n * 2); const sum = nums.reduce((a, b) => a + b, 0);

Template Literals

String interpolation, multi-line strings, and tagged templates

ES2015

📖 Explanation

Template literals use backticks (` `) instead of quotes and allow embedding expressions with ${expression}. They support multi-line strings natively and enable powerful "tagged template" patterns for custom string processing.

  • Embed any JavaScript expression: variables, math, function calls, ternary
  • Multi-line without escape characters
  • Tagged templates allow custom processing of template strings
JS — Template Literals
const name = "Alice"; const age = 30; const score = 95.5; // Basic interpolation console.log(`Hello, ${name}! You are ${age} years old.`); // Expressions inside ${} console.log(`Score: ${score.toFixed(1)}`); console.log(`Status: ${age >= 18 ? 'Adult' : 'Minor'}`); console.log(`2 + 2 = ${2 + 2}`); // Multi-line string const html = ` <div class="card"> <h2>${name}</h2> <p>Age: ${age}</p> </div> `; // Nested template literals const items = ['a', 'b', 'c']; console.log(`Items: ${items.map(i => `[${i}]`).join(', ')}`);

Destructuring

Unpack values from arrays and objects into distinct variables

ES2015

📖 Explanation

Destructuring lets you extract values from arrays or properties from objects into distinct variables in a single, concise statement. It's syntactic sugar that makes your code far more readable when working with complex data.

  • Array destructuring uses position; object destructuring uses property names
  • You can skip elements in arrays with commas
  • Default values prevent undefined when a value is missing
  • Rename variables with { original: newName }
  • Works in function parameters for clean APIs
JS — Array Destructuring
const [a, b, c] = [10, 20, 30]; console.log(a, b, c); // 10 20 30 // Skip elements const [first, , third] = [1, 2, 3]; // Default values const [x = 5, y = 7] = [1]; console.log(x, y); // 1 7 // Rest in array const [head, ...tail] = [1, 2, 3, 4]; console.log(head, tail); // 1 [2,3,4] // Swap variables! let p = 1, q = 2; [p, q] = [q, p]; console.log(p, q); // 2 1
JS — Object Destructuring
const user = { name: "Alice", age: 30, city: "NYC" }; // Basic const { name, age } = user; // Rename const { name: fullName, age: years } = user; // Default value const { city = "Unknown", role = "user" } = user; // Nested destructuring const { address: { street, zip } } = { address: { street: "Main St", zip: "10001" } }; // In function parameters function greet({ name, age = 0 }) { return `Hi ${name}, age ${age}`; } greet(user); // "Hi Alice, age 30"

Default Parameters

Set fallback values for function parameters

ES2015

📖 Explanation

Default parameters allow you to set a fallback value for a function argument if it's not provided or is undefined. They replace the old param = param || default pattern and are more explicit and safe.

  • Only triggers for undefined, not for null, 0, or ""
  • Default values can reference earlier parameters
  • Default values can be expressions or function calls
JS — Default Parameters
// Basic default function greet(name = "Guest") { return `Hello, ${name}!`; } console.log(greet()); // "Hello, Guest!" console.log(greet("Alice")); // "Hello, Alice!" console.log(greet(undefined)); // "Hello, Guest!" console.log(greet(null)); // "Hello, null!" (null ≠ undefined) // Expression as default function createId(prefix = "id", num = Math.random()) { return `${prefix}_${num.toString(36).slice(2)}`; } // Reference earlier param function range(start = 0, end = start + 10) { return { start, end }; } console.log(range(5)); // { start: 5, end: 15 }

Rest & Spread Operators

Collect remaining elements or expand iterables with ...

ES2015

📖 Explanation

Both use the same ... syntax but behave oppositely. Rest collects multiple elements into a single array (used in function params and destructuring). Spread expands an iterable (array, string, set) into individual elements.

  • Rest must be the last parameter in a function
  • Spread works with any iterable: arrays, strings, Sets, Maps
  • Spread creates shallow copies of objects and arrays
JS — Rest
// Rest in function params function sum(...nums) { return nums.reduce((a, b) => a + b, 0); } console.log(sum(1, 2, 3, 4)); // 10 function log(level, ...messages) { messages.forEach(m => console.log(`[${level}] ${m}`)); } log('INFO', 'Loading', 'Done'); // Rest in destructuring const [first, ...rest] = [1,2,3,4]; // first = 1, rest = [2,3,4] const { a, ...others } = { a:1, b:2, c:3 }; // a = 1, others = { b:2, c:3 }
JS — Spread
// Spread arrays const a = [1, 2, 3]; const b = [4, 5]; const combined = [...a, ...b]; // [1,2,3,4,5] // Clone array (shallow) const copy = [...a]; // Spread into function args Math.max(...a); // 3 // Spread objects const defaults = { theme: "dark", lang: "en" }; const settings = { ...defaults, lang: "fr" }; // { theme: "dark", lang: "fr" } // Spread string into chars const chars = [..."hello"]; // ['h','e','l','l','o'] // Convert Set to array const unique = [...new Set([1,1,2,3])]; // [1,2,3]

Enhanced Object Literals

Shorthand properties, methods, and computed keys

ES2015

📖 Explanation

ES6 added several shorthand features to object literals. If a variable name matches the property name, you can omit the value. Methods no longer need the function keyword. Property names can be computed dynamically with bracket notation.

JS — Enhanced Object Literals
const name = "Alice", age = 30; // Shorthand property (variable name = key name) const user = { name, age }; // same as { name: name, age: age } // Shorthand method const obj = { value: 42, getValue() { // no 'function' keyword return this.value; }, async fetchData() { }, // async shorthand *generator() { } // generator shorthand }; // Computed property names const key = "dynamic"; const prefix = "get"; const config = { [key]: "value", // { dynamic: "value" } [`${prefix}Name`]: "Alice", // { getName: "Alice" } [prefix + "Age"]: 30 // { getAge: 30 } }; // Getters and setters const person = { _name: "Bob", get name() { return this._name.toUpperCase(); }, set name(v) { this._name = v.trim(); } }; console.log(person.name); // "BOB"

Classes

Object-oriented programming with syntactic sugar over prototypes

ES2015ES2022 fields

📖 Explanation

ES6 classes are syntactic sugar over JavaScript's prototype-based inheritance. They provide a clean, readable syntax for creating objects with shared behavior. Classes support inheritance via extends, static methods, getters/setters, and private fields (ES2022).

  • constructor() — called when creating an instance with new
  • super() — must be called in child constructor before using this
  • static — methods on the class itself, not instances
  • #field — private field (ES2022), truly inaccessible from outside
JS — Classes & Inheritance
class Animal { // Class fields (ES2022) alive = true; #energy = 100; // private field constructor(name, sound) { this.name = name; this.sound = sound; } speak() { return `${this.name} says ${this.sound}!`; } eat(amount = 10) { this.#energy += amount; } get energy() { return this.#energy; } static kingdom() { return "Animalia"; } toString() { return `Animal(${this.name})`; } } class Dog extends Animal { constructor(name, breed) { super(name, "Woof"); // call parent constructor this.breed = breed; } fetch(item) { return `${this.name} fetched the ${item}!`; } speak() { // override parent method return `${super.speak()} *wags tail*`; } } const dog = new Dog("Rex", "Labrador"); console.log(dog.speak()); // "Rex says Woof! *wags tail*" console.log(dog.fetch("ball")); // "Rex fetched the ball!" console.log(Animal.kingdom()); // "Animalia" console.log(dog instanceof Animal); // true

Modules (import / export)

Split code into reusable files with named and default exports

ES2015

📖 Explanation

ES Modules allow you to split your JavaScript into separate files and share code between them. A module can export values and other modules can import them. There are two types: named exports (multiple per file) and default exports (one per file).

  • Modules run in strict mode automatically
  • Each module has its own scope — no global pollution
  • Use type="module" in script tag for browser modules
  • Dynamic import with import() for code splitting
JS — Modules
// ── math.js ────────────────────────── export const PI = 3.14159; export function add(a, b) { return a + b; } export function sub(a, b) { return a - b; } export default function multiply(a, b) { return a * b; } // ── main.js ────────────────────────── // Default import + named imports import multiply, { PI, add, sub } from './math.js'; // Namespace import (all as object) import * as math from './math.js'; math.add(1, 2); // Rename on import import { add as addNums } from './math.js'; // Re-export export { add, PI } from './math.js'; // Dynamic import (lazy loading) async function loadMath() { const module = await import('./math.js'); module.add(1, 2); }

Promises

Handle asynchronous operations without callback hell

ES2015

📖 Explanation

A Promise represents a value that may not be available yet — it will eventually resolve with a value or reject with an error. Promises have three states: pending, fulfilled, or rejected. They allow chaining with .then() and error handling with .catch().

  • Promise.all() — waits for ALL, rejects if any fails
  • Promise.allSettled() — waits for ALL, never rejects (ES2020)
  • Promise.race() — resolves/rejects with the first to settle
  • Promise.any() — resolves with the first fulfilled (ES2021)
JS — Promises
// Creating a Promise const fetchUser = (id) => new Promise((resolve, reject) => { setTimeout(() => { if (id > 0) resolve({ id, name: "Alice" }); else reject(new Error("Invalid ID")); }, 500); }); // Chaining .then() fetchUser(1) .then(user => { console.log(user); return user.id; }) .then(id => console.log("ID:", id)) .catch(err => console.error(err.message)) .finally(() => console.log("Done!")); // Promise combinators const p1 = Promise.resolve("A"); const p2 = Promise.resolve("B"); Promise.all([p1, p2]) .then(vals => console.log(vals)); // ["A","B"] Promise.allSettled([ Promise.resolve("ok"), Promise.reject("fail") ]).then(console.log);

Async / Await

Write asynchronous code that looks synchronous

ES2017

📖 Explanation

Async/await is syntactic sugar over Promises. An async function always returns a Promise. Inside it, await pauses execution until the Promise resolves, letting you write async code in a clean, synchronous style. Always use try/catch with await for error handling.

JS — Async / Await
// Basic async function async function getUser(id) { try { const res = await fetch(`/api/users/${id}`); const data = await res.json(); return data; } catch (err) { console.error("Failed:", err); } } // Arrow async function const delay = (ms) => new Promise(res => setTimeout(res, ms)); const run = async () => { console.log("Start"); await delay(1000); console.log("After 1s"); }; // Sequential vs Parallel async function sequential() { const a = await fetchA(); // waits const b = await fetchB(); // then waits } async function parallel() { // Both start simultaneously! Much faster const [a, b] = await Promise.all([fetchA(), fetchB()]); } // Async IIFE (top-level await alternative) (async () => { const result = await Promise.resolve("hello"); console.log(result); // "hello" })();

Symbol

Unique, immutable primitive values for object keys

ES2015

📖 Explanation

Symbol is a primitive data type whose values are unique and immutable. Every Symbol() call creates a completely new, unique value — even with the same description. Symbols are commonly used as unique property keys to avoid name collisions in objects and libraries.

JS — Symbol
// Always unique const a = Symbol("id"); const b = Symbol("id"); console.log(a === b); // false! // As object key (non-enumerable by default) const ID = Symbol("id"); const user = { [ID]: 42, name: "Alice" }; console.log(user[ID]); // 42 console.log(Object.keys(user)); // ["name"] — symbol hidden // Global symbol registry const s1 = Symbol.for("shared"); const s2 = Symbol.for("shared"); console.log(s1 === s2); // true (same registry key) // Well-known symbols — customize behavior class MyArray { static [Symbol.hasInstance](instance) { return Array.isArray(instance); } } console.log([] instanceof MyArray); // true const range = { from: 1, to: 5, [Symbol.iterator]() { let n = this.from; return { next: () => n <= this.to ? { value: n++, done: false } : { done: true } }; } }; console.log([...range]); // [1,2,3,4,5]

Generators & Iterators

Functions that can pause and resume execution

ES2015

📖 Explanation

Generator functions (marked with function*) can be paused with yield and resumed later. They return an iterator that lazily produces values on demand. Excellent for infinite sequences, lazy evaluation, and implementing custom iteration protocols.

  • yield — pauses and emits a value; caller receives it via .next()
  • yield* — delegates to another iterable/generator
  • Values can be passed back IN via gen.next(value)
JS — Generators
// Basic generator function* counter(start = 0) { while (true) { yield start++; } } const gen = counter(1); console.log(gen.next()); // { value: 1, done: false } console.log(gen.next()); // { value: 2, done: false } // Finite generator function* range(from, to, step = 1) { for (let i = from; i <= to; i += step) { yield i; } } console.log([...range(1, 10, 2)]); // [1,3,5,7,9] // yield* — delegate to another iterable function* combined() { yield* [1, 2]; yield* "AB"; yield 99; } console.log([...combined()]); // [1,2,'A','B',99] // Two-way communication function* adder() { let total = 0; while (true) { const n = yield total; if (n === null) break; total += n; } } const add = adder(); add.next(); // { value: 0 } — start add.next(10); // { value: 10 } add.next(5); // { value: 15 }

Map & Set

Advanced data structures with ordered keys and unique values

ES2015

📖 Explanation

Map is like an object but keys can be ANY type (objects, functions, etc.) and insertion order is preserved. Set stores unique values of any type. Both are iterable and have helpful methods.

  • Map vs Object: Map keys can be any type; Map is directly iterable; Map has .size
  • Set is like an array but all values are unique
  • Both maintain insertion order
JS — Map
const map = new Map(); // Keys can be ANY type map.set("name", "Alice"); map.set(42, "number key"); map.set(true, "bool key"); const objKey = {}; map.set(objKey, "object key"); console.log(map.get("name")); // "Alice" console.log(map.has(42)); // true console.log(map.size); // 4 // Initialize from entries const m2 = new Map([["a",1],["b",2]]); // Iterate for (const [key, val] of m2) { console.log(key, "->", val); } // Convert to array const entries = [...m2];
JS — Set
const set = new Set([1, 2, 2, 3, 3]); console.log(set); // Set {1, 2, 3} console.log(set.size); // 3 set.add(4); set.delete(1); set.has(2); // true // Remove duplicates from array const arr = [1,1,2,3,3,4]; const unique = [...new Set(arr)]; // [1, 2, 3, 4] // Set operations const a = new Set([1,2,3]); const b = new Set([2,3,4]); // Union const union = new Set([...a, ...b]); // Intersection const inter = new Set([...a].filter(x => b.has(x))); // Difference const diff = new Set([...a].filter(x => !b.has(x)));

WeakMap & WeakSet

Garbage-collectible collections for object keys

ES2015

📖 Explanation

WeakMap and WeakSet are like Map/Set but they hold weak references — if the key object is no longer referenced elsewhere, it can be garbage collected automatically. This prevents memory leaks. They are NOT iterable and don't have a .size property.

  • Keys must be objects (not primitives)
  • Not iterable — no forEach, no size
  • Perfect for: DOM node metadata, private data, caching
JS — WeakMap & WeakSet
// WeakMap — private data pattern const _private = new WeakMap(); class User { constructor(name, password) { this.name = name; _private.set(this, { password }); // truly private } checkPwd(pwd) { return _private.get(this).password === pwd; } } const u = new User("Alice", "secret123"); console.log(u.checkPwd("secret123")); // true // u.password → undefined (truly private) // WeakSet — track visited objects const visited = new WeakSet(); function process(obj) { if (visited.has(obj)) { return "Already processed!"; } visited.add(obj); return "Processed"; } const item = { data: 42 }; console.log(process(item)); // "Processed" console.log(process(item)); // "Already processed!"

Proxy & Reflect

Intercept and redefine fundamental object operations

ES2015

📖 Explanation

A Proxy wraps an object and intercepts operations on it via "traps" (get, set, has, deleteProperty, etc.). Reflect provides the default implementations of those operations. They power reactive systems like Vue 3's reactivity.

JS — Proxy & Reflect
// Validation proxy const createValidator = (target, schema) => new Proxy(target, { set(obj, key, value) { if (schema[key] && typeof value !== schema[key]) { throw new TypeError(`${key} must be ${schema[key]}`); } return Reflect.set(obj, key, value); } }); const user = createValidator({}, { name: "string", age: "number" }); user.name = "Alice"; // ✅ // user.age = "old" → TypeError: age must be number // Logging / observability proxy const observe = (obj) => new Proxy(obj, { get(t, k) { console.log(`GET ${k}`); return Reflect.get(t, k); }, set(t, k, v) { console.log(`SET ${k} = ${v}`); return Reflect.set(t, k, v); } }); const state = observe({ count: 0 }); state.count++; // GET count → SET count = 1

Optional Chaining & Nullish Coalescing

Safely access nested values with ?. and ??

ES2020

📖 Explanation

Optional chaining (?.) lets you access nested object properties without throwing if an intermediate value is null or undefined. Nullish coalescing (??) returns the right side only when the left side is null or undefined (unlike || which also catches falsy values like 0, "", false).

JS — Optional Chaining & Nullish Coalescing
const user = { name: "Alice", address: { city: "NYC" }, greet: () => "Hello!" }; // Optional chaining — no more "Cannot read property of undefined" console.log(user?.address?.city); // "NYC" console.log(user?.phone?.number); // undefined (no error!) console.log(user?.greet?.()); // "Hello!" console.log(user?.missing?.()); // undefined // With arrays const arr = null; console.log(arr?.[0]); // undefined // Nullish coalescing ?? vs || console.log(0 || "default"); // "default" (0 is falsy) console.log(0 ?? "default"); // 0 (0 is not null/undefined) console.log("" ?? "default"); // "" (empty string is ok) console.log(null ?? "default"); // "default" console.log(undefined ?? "default"); // "default" // Logical assignment operators (ES2021) let a = null; a ??= "value"; // a = "value" (only if null/undefined) let b = 0; b ||= 42; // b = 42 (only if falsy) let c = 1; c &&= 99; // c = 99 (only if truthy)

Tagged Template Literals

Custom processing of template strings with tag functions

ES2015

📖 Explanation

A tagged template is a template literal prefixed with a function name. The function receives the string parts and interpolated values separately, allowing custom processing — sanitizing HTML, i18n, SQL escaping, styled-components CSS, and more.

JS — Tagged Templates
// Tag function signature function tag(strings, ...values) { console.log(strings); // array of string parts console.log(values); // array of interpolated values return strings.reduce((r, s, i) => r + (values[i-1] || "") + s); } // HTML sanitizer tag function safe(strings, ...vals) { const escape = s => String(s) .replace(/&/g, "&amp;") .replace(/</g, "&lt;") .replace(/>/g, "&gt;"); return strings.reduce((r, s, i) => r + escape(vals[i-1] ?? "") + s); } const input = '<script>alert("xss")</script>'; console.log(safe`User input: ${input}`); // "User input: &lt;script&gt;alert..." // CSS-in-JS pattern (like styled-components) function css(strings, ...vals) { return strings.reduce((r, s, i) => r + (vals[i-1] ?? "") + s); } const color = "red", size = 16; const styles = css`color: ${color}; font-size: ${size}px;`;

for...of & for...in

Modern iteration over values and keys

ES2015

📖 Explanation

for...of iterates over the values of any iterable (arrays, strings, Maps, Sets, generators). for...in iterates over the enumerable keys of an object (including inherited ones). Use for...of for arrays; for...in for objects.

JS — for...of
// Arrays for (const num of [10, 20, 30]) { console.log(num); // 10, 20, 30 } // With index via entries() for (const [i, v] of ["a","b","c"].entries()) { console.log(i, v); // 0 "a", 1 "b"... } // Strings for (const ch of "hello") { console.log(ch); } // Map const m = new Map([["a",1],["b",2]]); for (const [k, v] of m) { console.log(k, v); } // Generator function* nums() { yield 1; yield 2; yield 3; } for (const n of nums()) console.log(n);
JS — for...in
const obj = { a: 1, b: 2, c: 3 }; // for...in — iterates keys for (const key in obj) { console.log(key, obj[key]); } // Filter inherited props with hasOwn for (const key in obj) { if (Object.hasOwn(obj, key)) { console.log(key); // own props only } } // ⚠️ Don't use for...in on arrays // Use for...of or forEach instead // Quick comparison const arr = ["x", "y", "z"]; // for...in on array → gets indices (strings!) for (const i in arr) console.log(i); // "0","1","2" // for...of on array → gets values for (const v of arr) console.log(v); // "x","y","z"
